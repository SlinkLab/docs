The `user:reset-password` command allows administrators to reset a user's password from the command line without knowing the current password. This is useful for recovering locked-out accounts or administrative password changes.

## Usage

```bash
docker exec -it slink slink user:reset-password --email=<email>
```

When `--password` is omitted, the command prompts for the new password with hidden input and asks you to confirm it. The password is never displayed on screen.

## Options

| Option | Alias | Required | Description |
| --- | --- | --- | --- |
| `--email` | | Yes | Email address of the user whose password to reset. If omitted, you are prompted. |
| `--password` | `-p` | No | Set the new password inline. If omitted, you are prompted (with confirmation). |

<Aside type="caution">
  Passing `--password` leaves the password visible in shell history and process listings. For sensitive environments, prefer the interactive prompt.
</Aside>

<Aside type="note">
  Unlike a user changing their own password in the app, this command does not require the current password. It is an administrative reset. The user can immediately sign in with the new password.
</Aside>

## Errors

| Message | Cause |
| --- | --- |
| `Error: User not found` | No user exists with the given email. |
| `Error: Passwords do not match.` | The password and confirmation entered at the interactive prompt differ. |
