The `user:create` command creates user accounts from the command line. This is useful for administrative tasks and automated setup.

## Usage

```bash
docker exec -it slink slink user:create \
  --email=<email> \
  --username=<username>
```

The command prompts for a password interactively unless you pass `--password`.

## Options

| Option | Alias | Required | Description |
| --- | --- | --- | --- |
| `--email` | | Yes | User's email address. |
| `--username` | | Yes | Username. |
| `--display-name` | | No | Display name. Defaults to the username. |
| `--password` | `-p` | No | Set the password inline. If omitted, you are prompted. |
| `--activate` | `-a` | No | Activate immediately instead of leaving the account pending approval. |

<Aside type="caution">
  Passing `--password` leaves the password visible in shell history and process listings. For sensitive environments, prefer the interactive prompt.
</Aside>

<Aside type="tip">
  When `USER_APPROVAL_REQUIRED=true`, users are created pending approval by default. Use `--activate` to create active users immediately.
</Aside>

## Related Commands

- [`user:activate`](/user-management/02-approve-user/) activates a pending user account
- [`user:grant:role`](/user-management/03-manage-user-permissions/) grants roles to users
- [`user:revoke:role`](/user-management/03-manage-user-permissions/) revokes roles from users

## Errors

| Message | Cause |
| --- | --- |
| `Error: Email already registered.` | The email is already in use. |
| `Error: Username already exist.` | The username is already taken. |
| `Error: Invalid email address.` | The email format is invalid. |
| `Error: Min 6 characters password` | The password is shorter than 6 characters. |

<Aside type="note">
  Accounts created via CLI are subject to the same validation rules as accounts created through the web interface.
</Aside>
